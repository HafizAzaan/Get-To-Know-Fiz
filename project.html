Book Exchange System

This project, titled “Book Exchange System”, is designed to facilitate book exchanges between users. The primary users are students who are either willing to lend their books (Lenders) or looking for books to read for entertainment or educational purposes (Borrowers). This project was developed by a team of three during our degree semester. The content on this page focuses solely on the Borrower’s perspective.

Above image is a use case of the project. There are two users, a Lender and a Borrower. Both Borrower and Lender share common modules such as Register, Login and Logout. Borrower has few modules where they can Brose Book, Request For Book, Return Book. Meanwhile, Lender has a fewer modules compared to Borrower. Lender has Register Book, Accept Borrow Book Request etc.

Both Borrower and Lender will be greeted with this homepage where some of the registered books being displayed here.

This is the page where user can choose whether to login as Borrower or Lender by clicking the radio button.

After logging in, Borrowers can view their borrowed books, read descriptions, and borrow books using the provided button.

After clicking 'Borrow', Borrowers fill in details to request the book, which is then sent to the Lender for approval.

(Optional) Borrowers can check the status of their book borrowing request to see if it has been approved by the Lender.

To explore the Lender's perspective, feel free to contact me.

Technologies Used: HTML, CSS, JavaScript Firebase

Ride Hailing with Admin

This project has been developed for my final year project. The aim of this project is to solve the problem of students struggling to find ride-hailing services from other students. Current solutions, like creating social media groups or posting announcements in random group chats, often fail because they get lost in too much information. This project proposes a new platform where students can easily find ride-hailing services offered by other students with just a few taps on their mobile devices. This platform will make it more convenient and ensure important information isn't missed. Here, only Passenger's and Admin's perspective will be provided.

The use case diagram above highlights the key components of this project, showcasing three main users: the Admin, the Driver, and the Passenger. Each user has a distinct role in the system’s operations. While both the Passenger and Driver can access all features via a mobile application, the Admin can only manage the system through an HTML page.

After installation, the user will see the Login Page and receive an error message for incorrect email or password.

If the user hasn't registered, they can go to this page to enter their information. Once registered, they can immediately log in.

Upon login as a Passenger, the user will be greeted with a page displaying their current location. There will also be a field where they can enter their desired destination.

This page provides location suggestions, exclusively within Malaysia. It displays a list of recommended places for the user to choose from, helping them easily select their desired destination within the country.

After entering the start and destination locations, the app will display a button to confirm the ride, along with the distance in kilometers and the fare.

Once the ride is confirmed, the app will send a notification to the driver. If the driver accepts the request, the user will see this page, which shows the driver's current location and provides an option to call the driver.

After the driver has completed the pickup and drop-off, the passenger will be provided with a "Finish" button. Upon clicking it, a page will pop up displaying the total price to be paid.

After the process is complete, the app will prompt the passenger to rate the ride. The rating will be visible to other passengers the next time they request a ride. (FINISHED FOR PASSENGER'S PERSPECTIVE)

Admin will be greeted by default page at the start before being able to do any action. Admin’s role is to control the Passenger and Driver.

Admin will see the total number of passengers and drivers through overview page. The number of blocked passengers and drivers also be displayed here.

Admin has the power to block or unblock Driver if getting any report about the behavior of Driver. Once Driver has been block, Driver no longer can log in their account and need to contact the Admin.

Technologies Used: Android (Flutter), Java, Firebase, Google Maps API

Toll Calculator System

The Toll Calculator project, developed during my internship at United Seino Transportation Sdn Bhd, addresses the needs of a logistics company specializing in truck transportation across Malaysia's highways. It provides accurate toll cost estimation for multiple drop-off locations, improving planning and budgeting. Traditional methods were slow and error-prone, risking client billing. This system, built with HTML, CSS, JavaScript, and the TollGuru API, offers an efficient, user-friendly solution to enhance operations.

This is the first page users will see, where they can view a map of Malaysia and fill in fields to calculate the toll prices for their journey.

Technologies Used: TollGuru API, HTML, CSS, JavaScript
